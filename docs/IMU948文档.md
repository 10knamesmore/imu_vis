## 一、产品概述

IM948 是辰翼电子科技自主研发的一款兼容蓝牙和串口的小尺寸十轴姿态传感器模块。

- 集成 ARM 32位DSP处理器、BLE5.3蓝牙、高精度加速计、陀螺仪、磁力计、温度气压计。
- 采用自主研发的姿态解算算法，强实时、高精度、稳定不漂移。
- 内置充电管理，超低功耗技术，并支持丰富的功能扩展。
- 工艺上采用4层阻抗板，沉金邮票孔接口。
- 蓝牙5.3无线技术，支持 Android、iOS、Windows、Linux 系统及其它BLE蓝牙设备连接使用。

适合机器人、飞控、游戏机、VR/AR、工控等产品的运动姿态检测、振动检测、动作捕捉、惯性轨迹捕获等解决方案使用。

## 二、模块轴向说明

模块出厂设置的默认轴向已在模块表面标示（即东北天坐标系）：

- 向右为 **X轴**
- 向前为 **Y轴**
- 垂直芯片向外为 **Z轴**

旋转方向按右手法则定义：右手大拇指指向轴向，四指弯曲的方向为绕该轴旋转的方向。

若模块默认轴向不符合需求，可通过 **PCB安装方向设置指令（0x20）** 进行更改。

## 四、模块管脚图

> 注意：所有信号IO管脚不要超过3.4V

### 引脚描述

| 引脚名 | 引脚号 | 类型   | 描述                                                                                                 |
| ------ | ------ | ------ | ---------------------------------------------------------------------------------------------------- |
| VSS    | 1/8/16 | PWR    | 电源负极，此3脚在模块内部短接                                                                        |
| VBUS   | 2      | PWR    | 充电输入电压，4.5-5.5V，充电电流可软件配置（注意①）                                                  |
| VBAT   | 3/15   | PWR    | 电源输入，2.5-4.5V，典型锂电池输入3.7V，第3脚和第15脚内部短接（注意②）                               |
| VDDIO  | 4      | PWR    | GPIO电压输入/输出端，2.5-3.4V；VBAT>3.1V时由内部LDO降为3.0V；VBAT≤3.1V时需将VBAT与VDDIO短接（注意③） |
| NC     | 5/6    | —      | 出厂内部测试，不用时悬空，不能接地                                                                   |
| ADIN   | 7      | INPUT  | 用户AD转换功能输入端口，最大输入电压≤3.4V                                                            |
| LED-B  | 9      | OUTPUT | 与模块内部蓝色指示灯短接，灯亮时输出低电平（注意④）                                                  |
| LED-R  | 10     | OUTPUT | 与模块内部红色指示灯短接，灯亮时输出低电平（注意④）                                                  |
| GPIO1  | 11     | I/O    | 用户自定义GPIO端口，输入输出状态可由用户设置                                                         |
| KEY    | 12     | INPUT  | 按键输入端口，低电平>2S后释放则关机；关机状态下低电平脉冲开机                                        |
| RXD    | 13     | INPUT  | 模块串口接收脚，高电平=VDDIO，115200bps，8N1                                                         |
| TXD    | 14     | OUTPUT | 模块串口发送脚，高电平=VDDIO，115200bps，8N1                                                         |

### 注意事项

**①** VBUS为充电输入（4.5-5.5V），接+5V时VBAT由内部充电管理降为4.2V输出。VBAT无外接电源时，VDDIO经内部LDO降为3.0V。串口与5V TTL系统通信需经过电压转换电路。

**②** VBAT为主电源输入（2.5-4.5V）。输入电压>3.1V时，保持VDDIO悬空即可。输入电压≤3.1V时，需短接VBAT和VDDIO，否则模块无法正常工作。

**③** VDDIO为模块IO电压（2.5-3.4V）。未外接电源且VBAT>3.1V时，由内部LDO主动降为3.0V输出。若VBAT≤3.0V或需要GPIO电压>3.0V，则需短接VDDIO与VBAT（此时VBAT应<3.4V）。

**④** 指示灯状态说明：

| LED-B（蓝色） | LED-R（红色） | 状态描述                             |
| ------------- | ------------- | ------------------------------------ |
| 亮2秒         | 亮2秒         | 设备刚开机                           |
| 灭            | 常亮          | 正在充电                             |
| 常亮          | 灭            | 已充满电                             |
| 单闪          | 灭            | 电量正常，蓝牙广播中                 |
| 双闪          | 灭            | 电量正常，蓝牙已连接                 |
| 灭            | 单闪          | 电量低于20%，蓝牙广播中              |
| 灭            | 双闪          | 电量低于20%，蓝牙已连接              |
| 三闪          | 灭            | 电量正常，当前串口或蓝牙通信带宽不足 |
| 灭            | 三闪          | 电量低于20%，当前通信带宽不足        |

**带宽不足解决办法：**

- a. 合理降低主动上报频率，减少订阅的数据量
- b. 检查蓝牙通信距离是否过远或存在干扰源

## 六、通信协议

IM948模块支持**串口TTL**和**蓝牙BLE5.3**两种通信方式。

串口TTL还支持 Modbus RTU 协议，详见：[IM948系列模块Modbus协议](https://www.yuque.com/cxqwork/lkw3sg/fifi44ron27lg73a?singleDoc#)

### 默认协议说明

**串口参数：** 115200bps、8数据位、1停止位、无校验位

**数据包格式：**

```
前导码 + 起始码 + 地址 + 长度 + 数据体 + 校验码 + 结束码
```

| 字段   | 串口 | 蓝牙 | 说明                                             |
| ------ | ---- | ---- | ------------------------------------------------ |
| 前导码 | ✅   | ❌   | 固定0x00（46个），用于唤醒可能处于睡眠状态的模块 |
| 起始码 | ✅   | ❌   | 固定 0x49                                        |
| 地址   | ✅   | ❌   | 串口通信地址，0-254，广播地址255                 |
| 长度   | ✅   | ❌   | 数据体的字节数                                   |
| 数据体 | ✅   | ✅   | 功能标签+内容，小端对齐格式                      |
| 校验码 | ✅   | ❌   | 从地址到数据体每字节数据的和校验                 |
| 结束码 | ✅   | ❌   | 固定 0x4D                                        |

> 蓝牙BLE本身已确保数据包完整性，因此蓝牙只有数据体部分。

## 七、蓝牙功能使用说明

IM948模块集成超低功耗蓝牙BLE5.3，向下兼容4.2、4.0等低版本，支持Android、iOS、Linux、Windows等支持BLE功能的主机设备直接连接。

调试推荐工具：安卓使用 **nRF Connect**，苹果使用 **LightBlue**。

安卓开发参考开源库：https://github.com/Jasonchenlijian/FastBle

### 1、蓝牙广播信息

广播信息包含设备名称（广播名称会包含固件版本）、MAC地址、信号强度等。

### 2、蓝牙连接后的服务

> 只需关注以下服务UUID：

**服务UUID：** `0000ae30-0000-1000-8000-00805f9b34fb`

| 特征       | UUID                                   | 说明                                |
| ---------- | -------------------------------------- | ----------------------------------- |
| 发送特征值 | `0000ae01-0000-1000-8000-00805f9b34fb` | 必须使用 **write no response** 方式 |
| 接收特征值 | `0000ae02-0000-1000-8000-00805f9b34fb` | 需要使能 **notify** 才能接收数据    |

> **注意：** 每次连接模块后，要尽快发送1次保持蓝牙连接指令（0x29），否则模块会在30秒后自动断开。发送数据注意使用 write no response 方式。

另外，模块还提供标准的电量指示服务（Battery Service）。
